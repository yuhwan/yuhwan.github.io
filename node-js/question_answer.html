          # require 함수

          아마도 require를 쓰면 뭔가 마법같은 일이 일어나서 코드 내부에서 라이브러리를 쓸 수 있다고 느끼실 겁니다. 하지만, 코딩에서 미스테리한것은 존재하지 않는다는 것을 기억하세요. 조금만 더 시간을 가지고 노력하면 모든 것을 이해할 수 있습니다. 다만, 지금 내가 어느 깊이까지 개념을 이해하고 싶은지 고민하면됩니다. 지금부터는 require 함수에 대해서 미스테리를 풀어보고자 합니다.

          const variable = require('/path/to/file');

          require 함수에 전달되는 파라미터는 불러오고자하는 파일의 경로입니다. require는 전달된 경로에 있는 자바스크립트 파일을 실행시키고 그 결과를 variable에 저장하는 역할을 수행합니다. 예를 들어, 'Hello Require' 문구를 출력하는 library.js 파일을 만들어보세요.

          // library.js
console.log('Hello Require');

          이제 다시 index.js 파일로 돌아가서 아래와 같이 require함수에 library.js 파일의 경로를 전달해보세요. Hello Require가 출력되는 것을 확인하실 수 있을 겁니다. /Users/dan/node-js/node1/library.js 처럼 여러분 컴퓨터의 library.js 파일의 절대 경로를 전달해도 동일하게 동작합니다. 여기서 주목해야할 점은 상대경로든 절대경로든 지금 여러분은 require에 파일의 경로를 여러분이 require에 전달했다는 것입니다.

          // index.js
require('./library.js');

          위에서 moment 라이브러리를 불러올때 우리는 경로가 아니라 모듈의 이름만 전달했습니다. Node는 경로가 아닌 이름만 전달되는 경우에는 기존에 자신이 알고있는 경로에서 해당 파일을 찾습니다. 그 경로는 module.paths라는 변수를 출력해서 확인할 수 있습니다. index.js 파일을 아래처럼 변경해서 결과를 출력해보세요.

          // index.js
console.log(module.paths);

          여러가지 경로들이 나올텐데, 주목할 것은 npm install을 실행했을때 만들어진 node_modules 폴더가 이안에 있다는 것입니다. 위에서 확인했던 것처럼, 이 폴더에는 moment라는 저희가 설치한 폴더가 존재합니다. 여기서 한가지 이상하게 느끼실 겁니다. moment는 폴더고 library.js는 자바스크립트 파일인데 require는 폴더를 가져오는건가? 이렇게 말이죠. 해답은 moment 폴더 내부에 있는 package.json 파일에 있습니다. require는 node_modules에서 라이브러리를 가져오는 경우에 해당이름을 가지고 있는 폴더의 package.json 파일을 살펴봅니다. 그 안의 여러 필드중에 main 필드의 값에 적힌 파일을 실행하게 됩니다. (직접 찾아보세요). 예를 들어, moment의 경우에는 아래와 같이, moment.js 파일을 실행하라고 되어있네요.

          ...
"license": "MIT",
"main": "./moment.js",
"name": "moment",
...

          아직 require가 하는 일을 모두 설명하지 못했지만, 이 정도면 답답함이 조금은 가셨을 것이라고 생각됩니다. 지금처럼 모르는 함수나 개념이 나왔을때, 뭔가 자동으로 되겠지라는 마음보다는 어떻게 돌아가는지 정도는 알고 넘어가자는 마음으로 공부를 하신다면, 보다 빨리 성장하실 수 있을 겁니다.

          # Triple equals

          자바스크립트에서는 2가지 종류의 비교 연산자 즉, == 그리고 === 이 존재합니다. 보통 다른 언어에서는 3개로 구성된 비교 연산자가 존재하지 않기 때문에 많은 사람들이 헷갈려합니다. 이 2가지의 차이에 대해서 이해하기 전에, 아주 간단하게 정리하고 넘어가도록 하겠습니다. 앞으로 코딩을 하실 때에는 반드시 3개로 구성된 비교 연산자 === 만 사용하시면 됩니다. 이제부터 그 이유에 대해서 알아봅시다.

          먼저 ===는 값과 데이터 타입 2가지 모두를 비교합니다. 여기서 데이터 타입이란 이번 강의 제일 처음에 다루었던 Primitive Data Type을 생각하시면 됩니다. 예를 들어, 아래와 같이 비교 하면 오른쪽 커맨트처리가 되어있는 값이 결과로 나오게 됩니다. 당연한 결과 입니다.

          'hello world' === 'hello world' // true (Both Strings, equal values)
true === true // true (Both Booleans, equal values)
77 === '77' // false (Number vs. String)
false === 0 // false (Different type and different value)

          이에 반해 == 연산자는 비교 전에 타입 변환(Type Coercion)을 시도합니다. 여기서 타입 변환이란 우리가 숫자 + 문자를 하면 자연스럽게 문자열로 숫자가 바뀌는 것을 의미합니다.

          const str = 100 + "times";
console.log(typeof str); // string

          이 때문에 우리가 의도하지 않은 방식의 결과들이 나오게 됩니다. 이런 결과를 코드상에서 의도할 수 도 있겠지만, 좋지 않은 코딩 방식입니다. 매번 더 많은 생각을 하게 만들고, 사실 결과도 예측하기 어렵기 때문입니다.

          77 == '77' // true
false == 0 // true
0 == "" // true
null == undefined // true

          다시 한번 정리하면, 자바스크립트에서 비교 연산을 수행할때는 반드시 === 연산자를 사용하시기를 바랍니다.

          # 스트링 인터폴레이션

          스트링 인터폴레이션은 대부분의 언어가 제공하는 기능이지만, 문법은 언어마다 조금씩 차이가 납니다. 자바스크립트의 문법을 보기 전에 왜 많은 언어에서 이 기능을 제공하는지 그 필요에 대해서 느껴보도록 합시다.

          var moment = require('moment-timezone');
var seoulTimezone = moment().tz('Asia/Seoul').format('YYYY-MM-DD HH:mm:ss');
console.log('Asia/Seoul:  ' + seoulTimezone);

          지금 예제에서는 큰 문제가 없어보지만, 긴 문자열 중간에 변수를 3개 혹은 그 이상 합치고 싶은 경우에는 +를 굉장히 많이 사용해야합니다. 자바스크립트에서는 이를 쉽게 해주는 문법을 제공하고 있습니다. 그리고 이 방법을 스트링 인터폴레이션이라고 부릅니다. 문법은 굉장히 쉽습니다. ' 혹은 "로 문자열을 시작하는 대신, Back quote (`)를 사용해서 스트링을 만들면 됩니다. 2개의 Back quote 사이에 오는것은 무조건 스트링으로 다뤄집니다. 만약, 중간에 변수를 넣고 싶다면 ${variable} 형태로 넣어주면 됩니다. 아래처럼 말이죠.

          var firstVariable = 'Hello';
var secondVariable = 100;
var thirdVariable = moment();
console.log(`\t${firstVariable}: You logged in ${secondVariable} times.\nCurrent Time: ${thirdVariable}`);
